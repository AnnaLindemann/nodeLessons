Задание 1

Создание и настройка проекта.

    1. Создание проекта Node.js

Откройте терминал или командную строку.
Перейдите в папку, где вы хотите создать проект.
Запустите команду `npm init -y`, чтобы создать новый проект Node.js. Это создаст файл `package.json`.

    2. Настройка package.json

Настройте "type": "module"` в package.json для использования import.

    3. Установка зависимостей

В том же терминале выполните команду `npm install express sequelize sequelize-cli`. Эта команда установит все необходимые пакеты:
• Express для создания сервера.
• Sequelize для работы с базой данных.
• Sequelize CLI для автоматизации задач.

    4. Инициализация Sequelize

После установки зависимостей выполните команду `npx sequelize-cli init`.

    5. Настройка Express

Создайте файл `app.js` в корне вашего проекта.
Внутри `app.js`:
• Подключите Express.
• Создайте простой сервер, который будет отвечать на запросы.
• Убедитесь, что сервер запускается и работает корректно.

    Задание 2

Настройка MySQL и config.

    1. Откройте MySQL Workbench.

    2. Откройте существующее подключение, либо же создайте новое.

Задание 3

Настройка Sequelize в проекте

    1. Создание файла для настройки Sequelize:

В корневой папке вашего проекта найдите или создайте папку `config`, если её ещё нет.
Внутри этой папки создайте новый файл под названием `db.js`. Этот файл будет использоваться для настройки и инициализации Sequelize, а также для подключения вашего приложения к базе данных MySQL.

    2. Импорт необходимых модулей:

В файле `db.js` импортируйте модуль `Sequelize` из пакета `sequelize`. Вам также понадобится модуль `config`, который загрузит параметры подключения к базе данных из конфигурационного файла `config.json`.

    3. Инициализация Sequelize:

Используя импортированный модуль `Sequelize`, создайте новый экземпляр `Sequelize`, передав параметры подключения к базе данных, такие как имя базы данных, имя пользователя, пароль и другие необходимые настройки.

    4. Экспорт подключения:

В конце файла `db.js` экспортируйте созданный экземпляр `Sequelize`, чтобы использовать его в других частях вашего проекта, например, при определении моделей или выполнении операций с базой данных.

    5. Интеграция Sequelize с Express:

Перейдите в файл `app.js` в корневой папке вашего проекта. Импортируйте ваш файл `db.js`, чтобы установить подключение к базе данных при запуске приложения.
Настройте инициализацию подключения к базе данных при запуске сервера. Убедитесь, что ваше приложение успешно подключается к базе данных при запуске.

Задание 4

Создание модели `Product`

    1. Создание файла для модели:

В папке `models`, которую вы создали ранее, создайте новый файл для модели. Назовите его, например, `product.js`. Этот файл будет содержать описание модели `Product`, представляющей товары или услуги в вашем приложении.

    2. Импорт необходимых модулей:

В файле `product.js` импортируйте модули `DataTypes` из пакета `sequelize` и ваш экземпляр `sequelize` из файла `db.js`. Эти модули понадобятся для определения структуры модели и её подключения к базе данных.

    3. Определение модели `Product`:

В файле `product.js` создайте модель `Product`, используя метод `sequelize.define`. Эта модель будет соответствовать таблице `Products` в базе данных.
Определите поля (столбцы) модели, например: `name` (название продукта), `price` (цена продукта) и `description` (описание продукта).
Укажите типы данных для каждого поля и установите, какие из них обязательны для заполнения, а какие могут быть уникальными или иметь значения по умолчанию.

    4. Экспорт модели:

В конце файла `product.js` экспортируйте модель `Product`, чтобы использовать ее в других частях вашего приложения, например, для создания или получения данных о продуктах.

Задание 5

Создание и настройка миграции для модели `Product`

    1. Генерация файла миграции:

Откройте терминал в корневой папке вашего проекта и выполните команду для создания новой миграции, которая будет называться, например, `create-product-table`. Эта команда создаст новый файл миграции в папке `migrations`.

    2. Открытие и редактирование файла миграции:

Перейдите в папку `migrations` и найдите новый файл миграции с именем, включающим метку времени и имя миграции.
Измените формат файла с `.js` на `.cjs`.
Откройте этот файл для редактирования.

    3. Определение структуры таблицы `Products`:

В функции `up` опишите структуру таблицы `Products`. Укажите поля таблицы, такие как `id`, `name`, `price`, и `description`. Укажите типы данных для каждого поля, отметьте обязательные поля и те, которые должны быть уникальными.

    4. Откат изменений (при необходимости):

В функции `down` опишите действия для отката изменений, внесенных миграцией. В данном случае это будет удаление таблицы `Products`, чтобы база данных вернулась к исходному состоянию, если потребуется отменить миграцию.

    5. Сохранение файла миграции:

Сохраните изменения в файле миграции. Теперь этот файл готов к выполнению, и вы сможете применить его для создания таблицы `Products` в базе данных.

    6. Применение миграции:

В терминале выполните команду для применения миграции, чтобы создать таблицу `Products` в базе данных. Убедитесь, что в вашей базе данных появилась новая таблица с нужной структурой.

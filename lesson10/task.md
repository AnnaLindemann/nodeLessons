Создание маршрута для обновления профиля пользователя

    1. Создайте маршрут для обновления профиля: 

В вашем Express приложении создайте защищенный маршрут, например, `/update-profile`, который позволяет аутентифицированному пользователю обновить свои данные (например, имя пользователя, email и т.д.).

Убедитесь, что этот маршрут защищен с помощью JWT, чтобы только аутентифицированные пользователи могли обновлять свой профиль.

    2. Обработайте данные, отправленные пользователем:

Внутри маршрута получите данные, отправленные пользователем через `req.body`.
Проверьте, что все необходимые поля присутствуют и корректны.

    3. Обновите данные пользователя:

Найдите текущего пользователя (его данные можно взять из `req.user`, которые были декодированы из JWT).
Обновите его данные в "базе данных", массиве или другом используемом вами хранилище).

    4. Отправьте ответ пользователю:

Верните пользователю обновленные данные или сообщение об успешном обновлении.

    5. Тестирование маршрута:

Проверьте маршрут с разными пользователями, убедившись, что данные корректно обновляются только для аутентифицированного пользователя.

Задание 2

Создание маршрута для получения списка пользователей с ограничением по ролям

    1. Создайте маршрут для получения списка пользователей:

В вашем Express приложении создайте новый защищённый маршрут, например, `/users`, который возвращает список всех пользователей в системе.

    2. Добавьте ограничение по ролям:

Убедитесь, что доступ к этому маршруту имеют только пользователи с ролью `admin`. Для этого используйте middleware, проверяющее роль пользователя (например, `authorizeRole('admin')`).
Если пользователь не является администратором, маршрут должен возвращать ошибку `403 Forbidden`.

    3. Возвращайте данные:

В случае успешного запроса (если пользователь имеет роль администратора) отправьте в ответе JSON с информацией о всех пользователях, которые есть в "базе данных".

    4. Тестирование маршрута:

Проверьте маршрут, используя JWT токен администратора и обычного пользователя, убедившись, что доступ к списку пользователей имеет только администратор.

Задание 3

Реализация маршрута для смены пароля

    1. Создайте защищённый маршрут для смены пароля:

В вашем Express приложении создайте новый маршрут, например, `/change-password`, который позволяет пользователю сменить свой пароль.
Этот маршрут должен быть доступен только для аутентифицированных пользователей (защищён с помощью JWT).

    2. Получите и проверьте данные:

В маршруте получите текущий пароль и новый пароль из `req.body`.
Сначала проверьте, что текущий пароль, введённый пользователем, соответствует тому, что хранится в "базе данных" (помните, что пароли могут быть захешированы с использованием bcrypt).

    3. Обновите пароль пользователя:

Если текущий пароль верен, захешируйте новый пароль с помощью bcrypt и обновите его в "базе данных".

    4. Отправьте ответ пользователю:

Верните сообщение об успешной смене пароля. Если текущий пароль неверен, верните ошибку.

    5. Тестирование маршрута:

Проверьте маршрут с правильными и неправильными паролями, убедившись, что новый пароль корректно обновляется только при вводе правильного текущего пароля.
